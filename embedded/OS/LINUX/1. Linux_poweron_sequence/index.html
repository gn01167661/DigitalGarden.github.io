    一般 Linux SoC 開機流程

    1. 硬體復位 (Reset)

    2. SoC 復位控制器將所有外設與時鐘重置至預設值。

    3. 確保系統處於已知初始狀態。

    4. Boot ROM (第 1 階段引導程式)

    5. 內建於 SoC 的不可變更 ROM。

    6. 功能：驗證下一階段映像完整性、初始化最基本硬體（片上 SRAM）、跳轉至外部載體。

    7. 第一階段引導 (FSBL/SPL)

    8. 存放於 eMMC/SDCard 或 SPI Flash。

    9. 功能：初始化 DRAM、電源與時鐘管理單元 (PMU)；載入並執行主引導程式。

    10. 主引導程式 (U-Boot)

    11. 提供環境變數、引導參數設定與網路啟動功能。

    12. 載入 Device Tree Blob (DTB) 與 Linux Kernel 映像，並將控制權交給內核。

    13. Linux Kernel 啟動

    14. 解壓縮並展開映像 (zImage/initramfs)。

    15. 掛載根檔系統 (Rootfs)。
    16. 啟動使用者空間 init 進程 (systemd、init 等)。

    TrustZone SoC 開機流程

    1. Boot ROM (BL1)

    2. Stage-1 Bootloader，執行於 EL3 Secure。

    3. 核心任務：初始化最基礎硬體（片上 SRAM、最小時鐘）、驗證並載入 BL2。

    4. BL2 (Stage-2 Bootloader)

    5. 位於外部閃存，執行於 EL3 Secure。

    6. 功能：完成 DRAM、外設、PMU 全面初始化；載入 BL31、BL32、BL33。

    7. BL31 (Secure Monitor)

    8. 執行於 EL3 Secure。

    9. 提供 Secure Monitor 呼叫介面，管理 Secure/Non-secure 世界切換。

    10. BL32 (Secure Payload, 可選)

    11. 執行於 EL1 Secure（如 OP-TEE）。

    12. 提供加解密、鑑權等可信服務。

    13. BL33 (Non-secure Payload)

    14. 執行於 EL2/EL1 Non-secure（通常為 U-Boot 或 UEFI）。

    15. 完成非安全世界引導，最終跳轉至 Linux Kernel。

    16. Linux Kernel 啟動

    17. 同一般流程：解壓、掛載 Rootfs、啟動 init 進程。

    差異比較

    條目 一般 Linux SoC TrustZone SoC
    階段數 4 階段 (ROM → FSBL/SPL → U-Boot → Kernel) 6 階段 (BL1 → BL2 → BL31 → BL32 → BL33 → Kernel)
    安全分界 無硬體隔離，所有程式執行於同一特權層 嚴格區分 Secure/Non-secure 世界，由硬體強制隔離
    韌體更新 ROM 固定，FSBL/SPL 及 U-Boot 可自由更新 BL1 ROM 固定，其他階段映像需簽章驗證後才能更新
    開發複雜度 簡單，專注於 DRAM 與 U-Boot 配置 複雜，需處理 ATF、OP-TEE、Secure Monitor 及多階段簽章流程
    適用場景 一般應用處理器、工業控制、嵌入式產品 手機 SoC、政務/支付安全晶片、需要硬體受信任執行環境 (TEE) 的專案
